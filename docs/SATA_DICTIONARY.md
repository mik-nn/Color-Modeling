# DATA_DICTIONARY.md

## 1. ProfileMetadata

**Описание:** Метаданные, извлечённые из имени файла профиля.

- `full_name` — полное имя файла без расширения (например: "BC_Lyve_P9000_mk_CanvasMatte")
- `brand` — бренд (обычно "BC")
- `series` — серия материала или чернил (Lyve, VibranceLuster, 600MT и т.д.)
- `printer` — модель принтера (P9000 и др.)
- `ink` — тип чернил / режим (mk, pk, PLPP260 и т.д.)
- `substrate` — материал подложки (CanvasMatte, WCRW, VibranceLuster и т.д.)
- `parsed_at` — дата и время парсинга (ISO 8601)

## 2. Measurement (структура одной записи / патча)

**Описание:** Данные по одному цветовому патчу.

### Обязательные / базовые поля:
- `SAMPLE_ID` — уникальный идентификатор патча
- `CMYK_C`, `CMYK_M`, `CMYK_Y`, `CMYK_K` — номинальное покрытие в процентах (0–100)
- `LAB_L`, `LAB_A`, `LAB_B` — колориметрические значения в пространстве CIELAB

### Спектральные данные (вариативные имена колонок):

Спектральные значения могут быть представлены в разных форматах:

- `SPECTRUM_380`, `SPECTRUM_390`, ..., `SPECTRUM_730`
- `SPECTRAL_NM380`, `SPECTRAL_NM390`, ...
- `nm_380`, `nm_390`, ..., `nm_730`
- `nm380`, `nm390`, ...
- `R_380`, `R_400`, ... (отражение)
- `reflectance_380` и подобные варианты

**Рекомендация:** При загрузке данных нормализовать все спектральные колонки в единый массив `spectra: number[]` и массив длин волн `wavelengths: number[]` (от 380 до 730 нм с шагом 10 нм).

### Дополнительные вычисляемые поля:
- `deltaE00` — цветовая разница ΔE00 относительно reference профиля
- `residuals` — остатки после применения физической модели (YNSN)
- `is_outlier` — флаг выброса

## 3. ProfileData

**Описание:** Полная структура данных по одному ICC-профилю.

- `metadata` — объект типа ProfileMetadata
- `raw` — массив объектов Measurement (исходные данные)
- `clean` — массив объектов Measurement (после очистки)
- `has_spectral` — boolean (есть ли спектральные данные)
- `patch_count` — количество патчей
- `average_deltaE_raw_clean` — средняя разница ΔE00 между raw и clean версиями (показатель качества очистки)
- `wavelengths` — массив длин волн (если спектральные данные присутствуют)

## 4. AnalysisResult (результат сравнения двух профилей)

- `reference_substrate` — имя референсного субстрата
- `target_substrate` — имя целевого субстрата
- `pearson_corr_lab` — корреляция Пирсона по LAB
- `pearson_corr_residuals` — корреляция Пирсона по остаткам
- `r_squared` — коэффициент детерминации линейной регрессии
- `slope_stability_score` — оценка стабильности относительных наклонов между каналами
- `mean_deltaE_after_correction` — средняя ΔE00 после применения коррекции
- `n_patches_used` — количество патчей, использованных для коррекции
- `linearity_confidence` — общая оценка линейности переноса (высокая / средняя / низкая)
